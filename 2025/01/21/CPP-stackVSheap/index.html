Four segments of Memory

Code (Text)

assigned to store instructions that need to be executed.

Static/Global

stores all the static or global variables, variables that are nor declared inside a function, and that have the whole lifetime of an application, they are accessible anywhere as long as the application is running.

Stack

is used to store all the information of function calls amd all the local variables

  • When our program starts, our operating system allocates some amount of reserved space, like OS allocates 1 MB of space as stack (not really allocation, maybe just reserve,say I will have 1 MB), but the actual allocation of the stack frame and the actual allocation of the local variables happens from the stack during runtime and if our call stack grows beyond the reserved memory for the stack, then this is called stack overflow and our program will crash. (common happen when bad recursion, program goes infinitely into recursion)
  • Stack does not grow during runtime.
  • When a function is called, it is pushed onto stack, when it finishes, it is popped out of the stack. It is not possible to manipulate the scope of a variable if it is on the stack
  • Another limitation is that if we need to declare a large data type, like an array as local variable, then we need to know the size of the array at compile time only. If we have to decide how large the array will be based on some parameter during runtime then it is a problem with stack.
  • Like allocating large chunks of memory( size can be decided on run time ) or keeping variable in the memory till the time we want, we have heap.

stack

Heap (Dynamic memory)

  • Unlike stack, application’s heap is not fixed. We can totally control how much memory to use from the heap, till what time to keep. Heap can grow as long as do not run out of memory on the system itself.(So we Need to be careful using the heap!)
  • Heap also called dynamic memory. (Not the same thing with the heap in data structure!)
  • Need to free by programmer
