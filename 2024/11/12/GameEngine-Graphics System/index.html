How to Control the Game

  • J/K: Press J or K to turn the camera left or right.
  • UP/DOWN/LEFT/RIGHT/N/M: Press to control the game object(Tree) move.
  • N/M: Press N or M to control the Tree back/forward.
  • 1/2: Change the post process effect.
  • 4/5/6: Change the FXAA level.

Introduction to the Graphics Module in My Game Engine

Overview

My project builds on the foundation of my previous assignments, where I developed a basic graphics module. I’ve extended this module, adding more rendering features.

Rendering Features Implemented

Lighting Support

The module now supports 3 lighting types, including spotlights, point lights, and directional lights. This allows for dynamic lighting within the game scene.

Figure 1. Lighting

Shadow Mapping

I’ve implemented a basic shadow map, adding shadows cast by objects in the scene. The shadow depends on the directional light.

Figure 2. Shadow

Skybox

A skybox is included to create the appearance of a background. You can easily create a custom skybox effect by adding your own skybox textures.

Figure 3. Skybox

Post-Processing Effects

I’ve integrated support for post-processing effects in the rendering pipeline, allowing for flexible customization.
If needed, you can simply add a pixel shader to implement specific effects, such as bloom, blur, screen distortion, and more. This module makes it easy to create effects without changing the entire pipeline.
The post-processing shader takes two input textures: the scene render output texture and the scene depth texture. By combining these two textures, various effects can be achieved.

Figure 4. Post-Processing

FXAA (Fast Approximate Anti-Aliasing)

I implemented FXAA, a post-processing-based anti-aliasing technique.

How to Use My Graphics Module

Fundamental Purpose: fill the framebuffer needed by Graphics module

The essence of my Graphics module is to fill the framebuffer every frame with rendering data updated in every frame. This requires a careful setup of essential resources in the game project, including effects, meshes, lights, and other necessary elements."

Figure 9. Framebuffer struct

Figure 10. Submit rendering data to Framebuffer

In simple terms, I’ve built a rendering pipeline where you only need to pass the required resources to the Graphics module without worrying about the specifics of the rendering process.

Initialization of Resources in Game Initialization function

Before rendering, the game project must initialize various resources. This setup enables the Graphics module to receive all the resources it needs to populate the framebuffer during each frame.

Figure 12. Create Mesh

Dynamic Framebuffer Updates Based on Game Logic

A common requirement is the need for dynamic framebuffer updates based on game logic. The current graphics module allows for flexible framebuffer data updates.

For instance: Effect Switching:

If two different effects are initialized, the game can pass one of these to the Graphics module based on game state , such as the player input.

How to Create a New Effect and Add It to the Scene Rendering ?

For mesh rendering effects or post-processing effects, you can simply create your own shader to implement the desired effect. Then, use the path to your shader to create a new effect.
All shaders that fit the forward rendering pipeline for rendering meshes are essentially considered to be the same type of effect. You can easily replace the pixel shader to implement various effects.
It is like if you’d like a solid color shader to replace the basic lighting shader. You just need to create another pixel shader and then you can create the effect in the same way with the basic lighting shader.
However, for more specialized effects, like more advanced shadows or advanced anti-aliasing techniques, it’s not as straightforward. These effects require changes to the rendering pipeline’s process flow to be implemented correctly.

Do you have to use My GameObject Class?

It’s not necessary; you can use your own GameObject. However, the GameObject data required by the Graphics module must include certain components, such as mesh, effect, and rigidbody data, and since the implementations of classes like Mesh and Effect are within my Graphics module, you’ll need to modify the code in your original GameObject class to include these components.
