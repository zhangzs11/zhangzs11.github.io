Overview of the Wireframe Rendering Process

Wireframe rendering is the process of representing 3D objects using only their edges, without filling the faces.

Wireframe rendering involves three primary steps: vertex transformation, clipping, and line rasterization.

Step1 : Vertex Transformation

Transform 3D object vertices into a 2D space where they can be displayed on the screen.

  1. Object Space:
    Vertices are defined relative to the object’s local coordinate system.
  2. World Space:
    Apply a model matrix to place the object in the 3D world.
  3. View Space (Camera Space):
    Transform vertices using a view matrix to position them relative to the camera.
  4. Clip Space:
    Apply a projection matrix (perspective or orthographic) to project the 3D points into a normalized space.
    In this space: x, y, z coordinates are within [-1, 1], (z maybe [0, 1] in D3D, [-1, 1] in OpenGL), Vertices outside this range are clipped.
  5. Normalized Device Coordinates (NDC):
    Divide x,y,z by the homogeneous w to normalize the coordinates.
  6. Screen Space:
    Map NDC coordinates to pixel positions on the screen using the viewport transformation.

Step2 : Clipping

Use the Sutherland-Hodgman algorithm to clip triangles against the boundaries of the view frustum.
Triangles fully inside the frustum remain unchanged.
Triangles partially inside the frustum are split into smaller triangles or polygons.

Step3 : Line Rasterization

Use the DDA (Digital Differential Analyzer) algorithm to compute the intermediate pixels along each edge:
Incrementally calculate pixel positions between the start and end points of each line.
Handle both shallow and steep slopes to ensure accurate rasterization.
