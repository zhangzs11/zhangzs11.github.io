virtual

When I use the virtual keyword, a virtual table is being created.
Once I create these particular objects or rather these types, anytime I have just one thing marked as virtual, we need a virtual table.
So what is going to happen is within this class somewhere in c++ is being defined, there will be something known as a vtable pointer, pointer to some table.
When I actually create objects using a pointer where I am dynamically creating these objects, we are going to look inside of this vtable for our base object. And it is going to have a list of all the different functions. Whenever we have a base object to do look up in this table and call the appropriate function.

Same thing for the drive, I am going to have a vtable pointer, it is going to point to something for the particular derived class here. When I have member function here, this is going to point to the correct instance of member function for our derived class here. Like say, hi. point back to this particular function and make sure we call this one here.

So any time you have virtual in your classes here


A vtable is created.
