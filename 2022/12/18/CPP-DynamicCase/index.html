Dynamic Casting

Dynamic casting is provided to us as some of like a safety, when we want to do a specific type of casting.
The C-style cast which is just when you write your new type in brackets before a variable.
Dynamic cast is a C++ style cast, it doesn’t extra work to make sure that the way we actually cast. It actually does some validation for us to ensure that cast is valid.
More, a dynamic cast is a quite a tool more like a function. It is evaluated at runtime, does extra work, because of that it does have an associated runtime cost
What dynamic cast does is specifically used for casts along the inheritance hierarchy. If I have a class that is a subclass of another class, and I want to either cast to the base type of from the base type to a derived type, that is where I can actually use a dynamic casting.

For example. we have an entity. And then we have a player class and an enemy class, they derive from the entity.
If I just want to cast a player object bak to an entity object, that’s easy a player has already the type entity, so that can be done implicitly no casting is necessary.
The other way around is where things get more complicated, If we have an entity instance and we want to cast it to a player. We have no way of knowing whether it’s a player. We just know it is en entity. IF we don’t use dynamic casting, the compiler has to either believe us ans if we are wrong like it was en enemy, we try and access data that is unique to player. Our program will probably crash.
Dynamic cast is actually used to validate that. So what happens is if we have an entity instance that is actually an enemy. And we try to cast it to a player using dynamic casting, that cast will fail and that dynamic cast will return a null pointer. We could use it to check to see if an object is a given type. IN other words, I can try to do a dynamic cast on that entity object to convert it to player, and check the return if that return null.
